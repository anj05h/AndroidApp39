# Photo Album App

This project is a photo management application for Android, allowing users to organize photos into albums, tag them with people and locations, and search through their collection.


### 1. Prompts and Interactions

The following prompts were used to guide the AI development process:

**Feature Implementation & Logic:**
*   "I am building a JavaFX/Android photo album application. I need to create the main 'Home' screen... automatically load any album and photo data saved from the previous session. The screen should simply list all the existing album names in plain text."
*   "From the home screen, I need to be able to manage my albums. Can you provide the logic and UI code to create a new album, delete an existing one, and rename an album? When I open a specific album, the view should change to display all the photos inside it as small thumbnail images."
*   "I need features to add a new photo to it or remove one. I also need a 'Photo Display' screen where I can see a single image at full size. On this screen, please include manual slideshow controls (like 'Next' and 'Previous' buttons) so I can click through the photos in the album one by one."

*   "Need a robust search feature for the photos based on their tags (Person and Location). It needs to be case insensitive, and have an option where you can search it as a pair or separately and an autocompletion where all the possible tags show up in a dropdown like menu."

*   "Help me implement a visible 'three dots' menu for albums and photos instead of the long-press gesture."

**UI & Refactoring:**
*   "Update the `AlbumAdapter` to handle button clicks instead of long clicks."
*   "How do I make a generic search activity that filters photos by 'person' and 'location' tags?"
*   "Design an `item_photo.xml` layout that includes a CardView, an ImageView for the photo, and a footer with a caption and a menu button."

**Debugging and Error Fixes:**
*   "Fix 'Default Activity not found' error in Android Studio configuration."
*   "My build is failing with 'Installed Build Tools revision 35.0.0 is corrupted'. How do I fix this?"
*   "Resolve incompatibility between Java 24 and Gradle 8.4 (Update to Gradle 8.14+)."
*   "Authentication error when trying to run the app: 'Symbolic ID already exists'. Help me reset the IDE state."
*   "Fix compilation error: `SearchActivity` is not abstract and does not override abstract method."

### 2. Code Components

**Generated / Refactored by AI:**
*   **Project Layout**: Most of the initial project layout and boilerplate was generated by AI.
*   **Adapters**: `AlbumAdapter.java` and `PhotoAdapter.java` (Logic for handling click interactions and RecyclerView binding).
*   **Layouts**: `item_album.xml` and `item_photo.xml` (UI structure for lists).
*   **Debugging Configuration**: Files like `gradle-wrapper.properties` and `settings.gradle.kts` were modified by AI to fix build issues.
*   **Vector Assets**: Helper icons like `ic_more_vert.xml`.
* Format the readme.md file to make it more readable and easier to understand.

**Written by Student:**
*   **Tagging System**: The entire logic for adding, deleting, and displaying "Person" and "Location" tags was implemented by me. This includes the validation logic to restrict types and the UI updates to show them on the photo display.
*   **Core Data Model**: `Album.java`, `Photo.java`, and `Tag.java` were designed and implemented by me to structure the application data.
*   **Search Logic**: The filtering algorithms in `SearchActivity` were written by me to handle complex multi-tag queries and autocompletion matching.
*   **Album Management**: The features for creating, renaming, and deleting albums, as well as the thumbnail grid view associated with displaying photos within an album, were implemented by me.

### 3. Debugging Support

The AI provided critical assistance in resolving environment errors that were preventing the app from running, specifically:
*   Identifying and removing corrupted Android SDK build tools.
*   Diagnosing a corrupt `.idea` project state and guiding a re-import.
*   Updating the Gradle Wrapper to match the local Java JDK version.
